Theory

Key concepts

₿
Regtest
A private local blockchain that only exists on your machine. No real Bitcoin, no internet — just a controlled sandbox for learning and development.
🖥
Node
A program (bitcoind) that holds a complete copy of the blockchain. Your two nodes are like two banks that each keep their own independent ledger.
⏳
Mempool
The waiting room for unconfirmed transactions. A transaction lives here until a miner includes it in a block. Nothing is final until then.
🔗
Consensus
Bitcoin's core rule: every node always adopts the longest valid chain. This is how decentralized agreement works — no authority needed, no vote required.

Prerequisites

ℹ️ This project does not work in PowerShell or CMD. You need WSL2 — Ubuntu inside Windows, free and official.
# Step 1 — PowerShell as Administrator wsl --install # Restart your computer, then open Ubuntu from Start menu
# Step 2 — Inside Ubuntu terminal sudo apt update && sudo apt upgrade -y sudo apt install -y git curl wget python3 wslu
✅ From now on, every command in this workshop runs inside the Ubuntu terminal — not PowerShell.

# Step 2 — Install Python 3 (git is included with Homebrew) brew install python@3.11 git --version # should show git version 2.x.x
✅ Open Terminal with ⌘ Space → type Terminal → Enter. Homebrew also installs git.

Installation

Three steps to get started

01
Clone the project
Download the code and make scripts executable.
cd ~ git clone https://github.com/Kairos-Tek/bitcoin-workshop.git cd bitcoin-workshop chmod +x start-dashboard.sh stop-dashboard.sh scripts/*.sh
02
Install Bitcoin Core
Run the installer for your OS. Linux/WSL2: reload PATH after.
# macOS ./scripts/install-mac.sh # Linux / WSL2 ./scripts/install-linux.sh source ~/.bashrc
03
Start nodes & dashboard
Launch both Bitcoin nodes, then open the dashboard in your browser.
./scripts/start-nodes.sh ./start-dashboard.sh # → opens http://localhost:18500

The key insight

What happens when we connect the nodes?

node2 discovers that node1's chain is 102 blocks longer than its own (which had 0). Without any human intervention, node2 discards its empty state and downloads node1's entire chain.

“Nodes always adopt the longest valid chain.” This is Bitcoin's core consensus rule — no authority, no vote, no administrator.
./scripts/connect-nodes.sh

Block anatomy

Inside a block

HEIGHT
Position in the chain (102) — sequential, never reused
HASH
64-char SHA-256d fingerprint — uniquely identifies this block
PREV HASH
Hash of the previous block — this link is what makes the chain tamper-proof
MERKLE ROOT
Fingerprint of all transactions — altering any tx invalidates the block
NONCE
Number miners iterate until the hash meets the difficulty target
TIMESTAMP
Unix timestamp — when this block was mined (approximate)
Tx 1 — Coinbase (mining reward)
50 BTC created from nothing. Every block has exactly one. No inputs — coins are born here.
Tx 2 — Payment (inputs → outputs)
1 BTC → node2. Change output (~49 BTC − fee) back to node1. Two outputs total.

Hands-on time

Explore on your own

./scripts/mine-blocks.sh [N] [node]
Mine N blocks on node 1 or 2. Try mining 5 on node2 when both are connected — watch node1 sync instantly.
./scripts/send-transaction.sh [amount] [from] [to]
Send BTC between nodes. Script pauses at mempool — inspect the pending tx before it is confirmed.
./scripts/connect-nodes.sh
Connect node1 ↔ node2 as P2P peers. Run after Demo A to trigger the consensus sync moment.
./scripts/reset.sh
Wipe all blockchain data and start fresh. Safe at any time — run start-nodes.sh afterwards to restart.
💡 Challenge: reset → Demo A → connect nodes → send 0.5 BTC from node2 to node1 → mine 1 block. Find the change output in block detail and verify both balances.

Summary

What we learned today

01
Regtest = private, instant, free. No real money involved — the perfect sandbox for learning Bitcoin internals.
02
Coinbase maturity rule: freshly mined coins take 100 more blocks before they can be spent.
03
Mempool = the waiting room. Transactions live there until a miner picks them up. Nothing is final before that.
04
Consensus in action: isolated nodes build separate chains. Connected nodes always converge on the longest one.
05
Every block = coinbase + transactions. Each transaction has inputs (where coins come from) and outputs (where they go).
